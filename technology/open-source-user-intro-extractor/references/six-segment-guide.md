# 六段叙事提取指引

面向**软件使用者**（非开发者）。从 README/docs 提取**项目信息**，不是写演讲稿。

提取时对照 README、docs 核对；未知标 `未找到`。依据记入 `user-intro-scratch.md`，不写入最终文档。  
**产出语言：** 最终文档默认**简体中文**（命令、URL、路径保留原文）。

叙事顺序：**定位 → 痛点 → 转变 → 差异 → 上手 → 信任**。

---

## 第 1 段：钩子

**提取：** 一句话定位 + 一个具体使用画面（2～3 句，可用「你」）。

| 问题 | 去哪找 |
|------|--------|
| 用户向定位？ | README 首段、官网、docs 首页 |
| 典型使用 moment？ | 教程、showcase、截图说明 |
| 目标用户是谁？ | README、VISION |

**反例：** 「高性能可扩展平台」  
**正例：** 「在 Telegram 里发消息，几秒内收到 AI 回复」

---

## 第 2 段：痛点

**提取：** 2～3 条具体痛点（现象 + 感受）及本产品如何回应。

| 问题 | 去哪找 |
|------|--------|
| 换工具/App 的摩擦？ | 产品定位、对比文档 |
| 对 SaaS 的顾虑？ | 隐私、数据、订阅相关 docs |
| 产品声称解决什么？ | README problem statement、FAQ |

勿编造；至少 2 条。

---

## 第 3 段：转变

**提取：**

| 要素 | 内容 |
|------|------|
| Before | 痛点展开 |
| After | 用上产品后的可感知结果 |
| Because | 用户语言解释原因 |

另提取：happy path（≤3 步）、**第一次成功**如何判定（docs 依据）。

---

## 第 4 段：差异化

**提取：** 与 SaaS、与其他方案的最强差异（各 1～2 点）；开源卖点（按需 1～2 条）。

| 维度 | 去哪找 |
|------|--------|
| 数据位置、账号、隐私边界 | privacy、self-hosted docs |
| 独特定位 | README、comparison |

用对比表或短 bullet，勿列超过 5 条。

---

## 第 5 段：降低行动成本

**提取：** 官方上手路径的事实信息。

| 问题 | 去哪找 |
|------|--------|
| 安装方式与命令 | README、install 文档 |
| 向导/配置步骤 | getting-started、onboard |
| 官方上手耗时（若有） | docs 原文 |
| 前置条件 | 运行时、API Key、OS |
| 如何确认成功 | status、dashboard 等 |
| 平台差异 | Windows/macOS/Linux docs |
| 排障入口 | troubleshooting、FAQ |

记录：≤3 步、每步说明与命令/链接、总耗时（有依据才写）。

---

## 第 6 段：信任收尾

**提取：**

- 适合谁（2～3 条）
- 不太适合谁（1～2 条，诚实）
- 社会证明（若有，1～2 项：Star、案例、社区）
- **单一主 CTA**（一个安装命令或一个入门 URL）
- 文档 / FAQ / 社区链接（次要，≤2 个）

---

## 可选节：日常能力摘要

用户任务语言，≤5 条：对话、渠道、模型、自动化、扩展等。

---

## 可选节：安全与隐私（用户向）

数据存放位置、外部 API 边界、一句 docs 中的安全建议。勿写威胁模型。
